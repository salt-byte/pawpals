## Step 2: 创建第 1 个新应用（重复 7 次）

> ⚠️ 以下步骤对 **每个 bot 重复一次**，只是名称和描述不同。

### 2.1 创建应用

1. 点击右上角 **「创建企业自建应用」**
2. 填写信息：
   - **应用名称**: `职业规划师` （第 1 个 bot 的名字，见上表）
   - **应用描述**: `OpenClaw 求职系统 - 职业规划 Agent`
   - **应用图标**: 可以用 emoji 对应的图片，或随便选一个

3. 创建成功后，进入应用详情页

### 2.2 开启机器人能力

1. 在左侧菜单找到 **「应用能力」→「机器人」**
2. 点击 **「启用机器人」**

### 2.3 配置权限

在左侧菜单找到 **「权限管理」**，搜索并开启以下权限：

**直接导入权限 JSON**（在权限管理页面找「批量开通」或导入按钮，粘贴以下内容）：

```json
{
  "scopes": {
    "tenant": [
      "im:message",
      "im:message:send_as_bot",
      "im:message.group_at_msg:readonly",
      "im:chat:readonly",
      "im:resource",
      "contact:user.base:readonly"
    ],
    "user": []
  }
}
```

> 💡 如果不支持 JSON 导入，就手动搜索上面的 scope ID 逐个勾选。

### 2.4 配置事件订阅

1. 在左侧菜单找到 **「事件订阅」**（或 **「事件与回调」**）
2. **请求方式** 选择 **「使用长连接接收事件」（WebSocket）**
   
   > ⚠️ 这个很重要！必须选 WebSocket，不要选 HTTP 回调

3. 添加以下事件：

| 事件名称 | 事件 ID |
|---------|---------|
| 接收消息 | `im.message.receive_v1` |
| 消息已读 | `im.message.message_read_v1` |

### 2.5 记录 App ID 和 App Secret

1. 回到 **「凭证与基础信息」** 页面
2. 记下：
   - **App ID**: `cli_xxxxxxxxxx`
   - **App Secret**: `xxxxxxxxxxxxxx`
3. 把这两个发给我（或者记在下面的表格里）

### 2.6 发布应用

1. 在左侧菜单找到 **「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写版本号（如 `1.0.0`）和更新说明
4. 提交发布
5. 如果是企业内部应用，可能需要管理员审批（如果你是管理员就可以自己批）

> 💡 如果你用的是个人版飞书，可能不需要审批直接就能用

## 常见问题

**Q: 权限页面找不到某个权限？**
→ 有些权限需要先启用「机器人」能力才会出现。确保 Step 2.2 已完成。

**Q: 发布后群里看不到机器人？**
→ 检查应用是否已通过审批。如果是测试环境，可以先在「版本管理」里添加测试企业/用户。

**Q: WebSocket 选项在哪？**
→ 「事件与回调」页面顶部，有「请求方式」的选项，切换到「使用长连接接收事件」。

**Q: 可以用之前那个应用吗？**
→ 之前那个（`cli_a92224c2e8b89ced`）可以保留作为其中一个 bot（比如当"职业规划师"），这样只需要再新建 6 个。
